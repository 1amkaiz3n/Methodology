# Methodology Recon di WordPress (Structured Flow)

Setelah Subdomain enumeration dan URL crawler, lanjut ke tahap recon WordPress.

## 0. Workflow Overview

1. Subdomain enumeration + crawling
2. Passive enumeration (fingerprinting)
3. WordPress endpoint discovery (WP REST API + sitemap)
4. User enumeration
5. Plugin & theme enumeration
6. XML-RPC analysis
7. WPScan deep scan (version, plugins, themes, users, vulnerabilities)
8. Attack surface mapping (WP REST API + plugin namespaces)
9. Exposure checks (files, directories, configs, backups)
10. Plugin-specific / feature-specific endpoint enumeration

## 1. Passive Enumeration

### Version & Fingerprint

```bash
# Get WordPress version

## META generator
curl -s https://blog.onevasco.com/ | grep -i generator

## CSS/JS version fingerprint (lebih akurat)
curl -s https://blog.onevasco.com/ | grep -oP 'ver=[0-9.]+' | head
curl -s https://blog.onevasco.com/ | grep wp-includes

## asset fingerprint
curl -s https://blog.onevasco.com/ | grep wp- | grep ver=
```

### Plugin Discovery

```bash
# Get Plugins
curl -H 'Cache-Control: no-cache, no-store' -L -ik -s https://blog.onevasco.com/support/article/pages/ | grep -E 'wp-content/plugins/' | sed -E 's,href=|src=,THIIIIS,g' | awk -F "THIIIIS" '{print $2}' | cut -d "'" -f2

## atau
curl -s https://blog.onevasco.com | grep -oE "/wp-content/plugins/[^\"']+" | sort -u
```


### Pingback / XSPA

```xml
<!-- Basic scan -->
<methodCall>
  <methodName>pingback.ping</methodName>
  <params>
    <param>
      <value><string>http://127.0.0.1:8000/</string></value>
    </param>
    <param>
      <value><string>https://blog.onevasco.com/things-to-do/calgary-canada</string></value>
    </param>
  </params>
</methodCall>
```

> Jika dalam respons Anda mendapatkan faultCode dan nilai lebih besar dari 0 (`<value><int>17</int></value>`), maka itu berarti port terbuka + Anda dapat memverifikasi ini dengan memeriksa log server Anda.

## 4. User Enumeration

```bash
# wpscan
wpscan --url https://blog.onevasco.com --enumerate u

# Curl
## Username Enumerate
curl -s https://blog.onevasco.com/wp-json/wp/v2/users/ | jq '.[].slug'

## Cek user ID mapping
curl -s https://blog.onevasco.com/wp-json/wp/v2/users/ | jq '.[] | {id, slug, name}'

## Cek posts untuk semua author
for id in 258066334 258066338 231591501 258066328 258065771 258066327 258066337; do
  echo "=== User ID: $id ==="
  curl -s "https://blog.onevasco.com/wp-json/wp/v2/posts?author=$id&per_page=5" | jq '.[] | {title: .title.rendered, date: .date, link: .link}'
  echo ""
done

## Dapatkan semua user dengan detail
curl -s "https://blog.onevasco.com/wp-json/wp/v2/users?per_page=100" | \
  jq -r '.[] | "\(.id):\(.slug):\(.name)"'
```

## 6. API Surface Mapping

```bash
# =========================
# BASIC REST API ENUMERATION
# =========================

# Dump semua REST API root metadata (routes, namespaces, schema exposure)
curl -s https://blog.onevasco.com/wp-json/ | jq
# → tujuan: lihat full attack surface WP REST API (endpoint discovery awal)

# Filter endpoint user (potensi info leak / enumeration)
curl -s https://blog.onevasco.com/wp-json/ | jq | grep "/wp/v2/users"
# → cek apakah user endpoint exposed (risk: user enumeration)

# Filter endpoint posts (konten exposure + draft/leak check)
curl -s https://blog.onevasco.com/wp-json/ | jq | grep "/wp/v2/posts"
# → cek apakah ada endpoint post accessible (risk: data leakage / private posts)

# Filter endpoint comments (spam vector / stored XSS surface)
curl -s https://blog.onevasco.com/wp-json/ | jq | grep "/wp/v2/comments"
# → cek comment API (risk: XSS, unauthorized comment read/write)


# =========================
# AUTH & CONFIG SURFACE
# =========================

# Inspect WP REST base endpoint (auth requirement check)
curl -s https://blog.onevasco.com/wp-json/wp/v2/
# → cek apakah endpoint butuh auth atau kebuka (risk: unauthenticated access misconfig)

# =========================
# ROUTE ENUMERATION
# =========================

# List semua available REST routes (full attack surface mapping)
curl -s https://blog.onevasco.com/wp-json/ | jq '.routes | keys[]'
# → penting untuk mapping endpoint tersembunyi / plugin API exposure

# =========================
# SENSITIVE ENDPOINT CHECK
# =========================

# Test settings endpoint (high value target)
curl -s https://blog.onevasco.com/wp-json/wp/v2/settings
# → harusnya 401/403
# → kalau 200: CRITICAL misconfig (site config leak / potential takeover vector)

# =========================
# NAMESPACE ENUMERATION
# =========================

# List semua namespaces (core + plugin + theme APIs)
curl -s https://blog.onevasco.com/wp-json/ | jq '.namespaces'
# → identifikasi plugin aktif + custom API surface (attack surface expansion)
```
