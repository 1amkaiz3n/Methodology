# 🔷 ANALISA REFLECTED INPUT (PRAKTIS XSS METHODOLOGY)

## 🎯 Pertanyaan utama (SELALU TANYA INI)

> Input user ini muncul di mana, dan jadi apa di response?

---

## 🔥 REFLECTED CONTEXT (DAN LANGKAH NYA)

---

## 1. 🟡 HTML BODY

```html
<div>test123</div>
```

### 🔍 Ciri:

* input muncul langsung di halaman

### 🎯 Cek:

* apakah `< > & "` di-escape?
* apakah input mentah masuk HTML?

### ⚡ Risiko:

* tinggi kalau tidak di-encode

---

## 2. 🟠 ATTRIBUTE

```html
<input value="test123">
<a href="/search?q=test123">
```

### 🔍 Ciri:

* input masuk ke atribut HTML

### 🎯 Cek:

* bisa keluar dari `""` atau `'`?
* ada event handler di elemen itu?

### ⚡ Risiko:

* medium → high (tergantung escape)

---

## 3. 🔴 JAVASCRIPT STRING

```js
var q = "test123";
```

### 🔍 Ciri:

* input masuk ke JS variable

### 🎯 Cek:

* bisa break quote?
* ada dipakai lagi di logic JS?

### ⚡ Risiko:

* tinggi (kalau raw JS context)

---

## 4. 🟣 JSON / OBJECT JS

```js
window.data = { query: "test123" }
```

### 🔍 Ciri:

* data dari backend dipakai di frontend

### 🎯 Cek:

* dipakai untuk DOM render?
* ada `innerHTML` setelahnya?

### ⚡ Risiko:

* medium → high (kalau dipakai di DOM)

---

## 5. 🔵 API RESPONSE (JSON)

```json
{ "search": "test123" }
```

### 🔍 Ciri:

* input tidak langsung di HTML

### 🎯 Cek:

* frontend pakai render apa?
* ada `dangerous HTML render`?

### ⚡ Risiko:

* tergantung frontend (ini sering jadi DOM XSS)

---

## 6. 🔥 SCRIPT CONTEXT

```html
<script>
  var x = "test123";
</script>
```

### 🔍 Ciri:

* input masuk langsung ke JS block

### 🎯 Cek:

* bisa keluar dari string?
* ada eval / dynamic execution?

### ⚡ Risiko:

* SANGAT TINGGI

---

## 7. ⚫ EVENT HANDLER

```html
<button onclick="search('test123')">
```

### 🔍 Ciri:

* input masuk ke HTML event

### 🎯 Cek:

* bisa break function parameter?
* ada quote issue?

### ⚡ Risiko:

* tinggi

---

## 8. 🟢 DOM INJECTION (FRONTEND)

```js
el.innerHTML = input;
```

### 🔍 Ciri:

* input masuk setelah page load

### 🎯 Cek:

* React / Vue / Angular render?
* ada sanitizer?

### ⚡ Risiko:

* sangat sering jadi DOM XSS

---

## 9. 🔵 URL / REDIRECT

```html
<a href="/redirect?url=test123">
```

### 🔍 Ciri:

* input mengontrol navigasi

### 🎯 Cek:

* open redirect?
* dipakai di `window.location`?

### ⚡ Risiko:

* medium (bisa chaining attack)

---

## 10. 🔴 ERROR MESSAGE

```html
Error: test123 not found
```

### 🔍 Ciri:

* input muncul di error page

### 🎯 Cek:

* encoded atau tidak?
* bisa inject HTML?

### ⚡ Risiko:

* sering underestimated (kadang XSS di sini)

---

## 11. 🟣 TEMPLATE ENGINE

```html
<h1>{{query}}</h1>
```

### 🔍 Ciri:

* server-side rendering

### 🎯 Cek:

* encoding server?
* SSTI possible?

### ⚡ Risiko:

* bisa XSS atau SSTI (high impact)

---

## 🔥 FLOW METHODOLOGY (YANG HARUS DIHAFAL)

```text
Input ditemukan
   ↓
Reflected atau tidak?
   ↓
Masuk context apa?
   ↓
HTML / JS / DOM / Attribute / JSON?
   ↓
Encoded atau tidak?
   ↓
Dipakai di sink atau tidak?
   ↓
Exploit possible atau tidak?
```

---

## 💡 INTI SIMPEL (BIAR CEPAT PAHAM)

* HTML → fokus tag injection
* Attribute → fokus break `" "`
* JS → fokus break string + execution
* DOM → fokus frontend render
* JSON → fokus ke frontend usage
* Error → sering hidden XSS
* Template → bisa XSS atau SSTI

---

## ⚠️ KESALAHAN UMUM

❌ cuma lihat “reflected”
✔ harus lihat “context + sink + flow”
