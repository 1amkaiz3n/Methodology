# 🔷 DOM XSS & JavaScript Execution Sinks

## 🟡 DOM Manipulation Sinks

Dipakai untuk memasukkan HTML langsung ke DOM (sering jadi sumber DOM XSS):

* `innerHTML`
* `outerHTML`
* `insertAdjacentHTML`
* `document.write`

---

## 🔴 JavaScript Execution Sinks

Dipakai untuk mengeksekusi string sebagai JavaScript (high risk):

* `eval()`
* `Function()`
* `setTimeout(string)`
* `setInterval(string)`

---

## 💡 Konsep Dasar

Jika user input masuk ke salah satu sink di atas tanpa sanitasi, maka itu menjadi **area yang menarik untuk diuji lebih lanjut**.

---

## 🔷 Pertanyaan Inti yang Harus Dijawab

> “Input user ini mengalir ke mana?”

### Alur yang harus dipahami:

* Apakah masuk backend lalu diproses?
* Apakah kembali sebagai HTML response?
* Apakah disisipkan ke JavaScript?
* Apakah dipakai untuk DOM manipulation?

---

# 🔷 Sink → Payload Logic Mapping (XSS Hunting)

Tujuan:

> Menentukan cara testing berdasarkan jenis sink yang ditemukan

---

## 🟡 1. `innerHTML`, `outerHTML`, `insertAdjacentHTML`, `document.write`

### 👉 Context: HTML Injection

### 🔥 Logic:

Input akan diparse sebagai HTML oleh browser.

### 🧠 Yang diuji:

* apakah bisa inject tag HTML
* apakah karakter `< >` di-encode
* apakah struktur HTML bisa di-break

### 🎯 Fokus:

> kontrol rendering HTML di DOM

---

## 🔴 2. `eval()`

### 👉 Context: JavaScript Execution

### 🔥 Logic:

Input dieksekusi langsung sebagai kode JavaScript.

### 🧠 Yang diuji:

* apakah bisa keluar dari string context
* apakah input langsung dieksekusi

### 🎯 Fokus:

> full JavaScript execution

---

## 🔴 3. `Function()`

### 👉 Context: JavaScript Execution

### 🔥 Logic:

String dijadikan body function.

### 🧠 Yang diuji:

* injection ke function scope
* kontrol alur eksekusi JS

### 🎯 Fokus:

> arbitrary JS execution

---

## 🟠 4. `setTimeout("string")` / `setInterval("string")`

### 👉 Context: Delayed JavaScript Execution

### 🔥 Logic:

String dieksekusi sebagai JavaScript setelah delay.

### 🧠 Yang diuji:

* string injection
* breaking quote context
* execution context escape

### 🎯 Fokus:

> delayed JS execution

---

## 🟡 5. DOM Framework (React / Vue / SPA)

### 👉 Context: Client-side rendering

### 🔥 Logic:

Data dari API dirender ulang ke DOM oleh JavaScript.

### 🧠 Yang diuji:

* apakah ada sanitization (misalnya DOMPurify)
* apakah raw HTML masuk ke DOM

### 🎯 Fokus:

> client-side DOM XSS

---

## 🟣 6. `document.write`

### 👉 Context: Full document rewrite

### 🔥 Logic:

HTML ditulis langsung ke document sebelum rendering selesai.

### 🧠 Yang diuji:

* apakah script ikut tereksekusi
* timing injection saat page load

### 🎯 Fokus:

> full page HTML injection

---

## 🔵 7. URL / Navigation Sinks (`location`, `href`, redirect`)

### 👉 Context: Navigation control

### 🔥 Logic:

Input mengontrol alur URL atau redirect.

### 🧠 Yang diuji:

* open redirect
* kontrol flow aplikasi

### 🎯 Fokus:

> navigation hijack / chaining attack

---

# 🔷 QUICK DECISION TREE

```text
Sink ditemukan
   ↓
Kategori sink?
   ├── HTML sink → test HTML injection
   ├── JS sink → test JS execution break
   ├── DOM sink → test frontend rendering flow
   ├── URL sink → test redirect / flow control
   ↓
Check encoding / sanitization
   ↓
Check context exploitability
```

---

# 💡 INTI KONSEP

* HTML sinks → fokus ke struktur HTML (`<tag>`)
* JS sinks → fokus ke string breaking + execution
* DOM sinks → fokus ke frontend rendering flow
* URL sinks → fokus ke kontrol navigasi

---

# ⚠️ COMMON MISTAKE

❌ Menganggap semua sink punya cara exploit yang sama
✔ Setiap sink = konteks + teknik testing yang berbeda

---

# 🔥 KESIMPULAN

Ini sudah jadi **framework mental XSS yang benar**:

> Input → Context → Sink → Encoding → Exploitability
